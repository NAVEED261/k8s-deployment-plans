Project 2 — K8s Planning

Kubernetes Deployment Plans
for AI-Native Applications

Two detailed infrastructure plans, zero code. The whole point was to think through how you'd actually run these systems — who can access what, what happens when a pod crashes, how secrets rotate, all of it.

Two scenarios covered

AI Task Manager
UI, backend APIs, a task agent, and a notification service. Four namespaces, six secrets, proper RBAC so the agent only reads what it needs.
AI Employee — OpenClaw
Personal AI worker with access to emails, calendar, and code execution. Security was the whole conversation — gVisor sandbox, Vault, default-deny networking.

What each plan covers

Namespaces for logical isolation
Deployments and StatefulSets
ClusterIP, NodePort, LoadBalancer
CPU and memory sizing per service
ConfigMaps for environment config
Secrets with rotation schedules
RBAC roles and service accounts
Inter-service DNS and NetworkPolicy

Reusable skill

K8 Planning Skill
A reusable agent skill for any future project. Give it a list of services, a security level, and expected traffic — it walks you through namespace design, workload classification, service types, and a full security checklist. Includes three eval test cases for Anthropic's Skill Creator.
